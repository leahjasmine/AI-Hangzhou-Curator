# AI杭州文旅策展人

本项目是浙江树人学院《人工智能》课程大作业，是一个基于大模型的杭州个体化文旅定制网站。

## 项目简介

AI杭州文旅策展人以智慧文旅为主题，base 杭州，面向大学生、本地居民与外地游客，根据用户的审美偏好、出行心情、预算和时间，生成个性化路线、杭州文化问答、Citywalk 智能规划和文旅灵感卡片。

系统强调杭州本地文化 IP 与现代审美表达，将西湖、良渚玉琮、京杭大运河、灵隐飞来峰、龙井茶园、钱塘江潮、亚运莲花等文旅资源转化为结构化、可交互、可展示的 AI 文旅策展体验。

## 已实现功能

1. 网站首页框架
   - 顶部导航、主视觉区、功能区、AI 策展交互区、杭州文化 IP 灵感区与项目说明区。

2. 中英双语切换
   - 页面标题、导航、按钮、模块说明、输入提示均支持中文 / English 一键切换。

3. AI 文旅交互区
   - 支持路线生成、Citywalk、文化问答、灵感卡片四种模式。
   - 配置 API Key 后调用 OpenAI 兼容格式大模型接口。
   - 未配置 API Key 时自动使用本地演示回复，方便课堂展示。
   - 首页主视觉卡片、四个核心功能卡、六个杭州文化 IP 卡和示例成果卡均可点击，会自动跳转到 AI 策展台、切换对应功能并填入示例问题。

4. 中文交互文字样式
   - 参考现代粗黑体、字块化标题、色块标签、策展卡片式文案节奏。
   - 适合后续继续加入更细的视觉审美设计。

5. 杭州地标动画形象
   - 在首页加入“杭州”城市字标、雷峰塔、莲花、西湖水波与几何小物件组成的 SVG 动画形象。
   - 使用分层弹性入场、循环漂浮、水波流动和 hover 回弹，作为网页中的杭州城市参考形象。

6. 杭州文旅 IP 形象
   - 设计“杭小游”作为网页 IP：手捧龙井茶，采用 Q 版矮胖轮廓、粗黑手绘描边和多巴胺高饱和色块。
   - 支持眨眼、呼吸、挥手、茶气上升和周围装饰漂浮动画。

## 技术栈

- 前端：HTML5、CSS3、JavaScript
- 后端：Node.js、Express
- AI 能力：OpenAI 兼容格式大模型 API / 本地演示回复
- 视觉风格：江南现代文旅风、玻璃拟态、低饱和、城市策展页面、中文粗黑体交互区

## 运行方式：怎么查看完整功能网站

### 1. 安装依赖

第一次运行项目时，在项目根目录执行：

```bash
npm install
```

### 2. 启动项目

```bash
npm start
```

看到终端输出类似下面内容，说明服务已经启动：

```text
AI杭州文旅策展人已启动：http://localhost:3000
AI模式：本地演示模式
```

### 3. 打开网站

在浏览器访问：

```text
http://localhost:3000
```

不要直接双击 `public/index.html` 打开，因为 AI 问答需要通过 Node/Express 的 `/api/ai` 接口调用，必须先运行 `npm start`。

### 4. 检查 AI 接口是否正常

浏览器打开：

```text
http://localhost:3000/api/health
```

如果看到 JSON 内容，说明后端接口正常。例如：

```json
{
  "ok": true,
  "mode": "local-demo",
  "model": "deepseek-chat"
}
```

## 大模型 API 配置：怎么接入所需 API

项目支持 OpenAI 兼容格式的大模型接口。最简单方式是使用 DeepSeek/OpenAI 兼容接口。

### 1. 新建 `.env` 文件

把 `.env.example` 复制一份，改名为 `.env`。

Windows 终端可以执行：

```bash
copy .env.example .env
```

### 2. 填写 API 配置

打开 `.env`，填写：

```text
AI_API_KEY=你的API密钥
AI_API_URL=https://api.deepseek.com/chat/completions
AI_MODEL=deepseek-chat
PORT=3000
```

说明：

- `AI_API_KEY`：你的模型平台 API Key。
- `AI_API_URL`：OpenAI 兼容格式的聊天补全接口地址。
- `AI_MODEL`：模型名称，例如 `deepseek-chat`。
- `PORT`：本地端口，默认 `3000`。

### 3. 重启服务

修改 `.env` 后必须重启：

```bash
npm start
```

如果终端显示：

```text
AI模式：远程模型 deepseek-chat
```

说明已经进入真实 API 模式。

### 4. 无 API Key 时的情况

如果没有填写 `AI_API_KEY`，系统会自动使用内置本地演示模型，方便课堂展示。此时 AI 策展台仍然可用，但回答是预设演示内容，不是真实大模型生成。

### 5. API 报错排查

如果页面 AI 输出区出现“接口提示”，常见原因如下：

- `401` / `Unauthorized`：API Key 错误或没权限。
- `404`：`AI_API_URL` 或 `AI_MODEL` 写错。
- `429`：额度不足或请求过快。
- `fetch failed`：网络无法访问接口地址。

先访问 `http://localhost:3000/api/health` 确认服务正常，再检查 `.env`。
